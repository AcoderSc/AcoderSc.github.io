在linux集群服务器上部署hadoop

  1. 配置基础环境

1.1下载和安装jdk

使用rpm -e –nodeps + 文件名 可进行卸载

在usr目录下，创建java文件夹，发现该用户没有权限，使用sudo 命令

将下载好的linux版本的jdk8 移动至该文件夹下，并进行解压。如下：使用使用WinScp上传文件至linux虚拟机，主机名是虚拟机的ip地址，可用ifconfig -a 命令查看。

完成上传后，使用sudo tar -zxvf jdk-8u181-linux-x64.tar.gz 进行解压，

1.2配置java环境变量

使用 sudo vim /etc/profile 命令编辑文件

在文件中新增如下内容

JAVA_HOME=/usr/java/jdk1.8.0_181

JRE_HOME=/usr/java/jdk1.8.0_181/jre

CLASS_PATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$JRE_HOME/lib

PATH=$PATH:$JAVA_HOME/bin:$JRE_HOME/bin

export JAVA_HOME JRE_HOME CLASS_PATH PATH

之后键入:wq保存退出，使用source /etc/profile 命令使etc/profile修改后立即生效。

输入java -version，显示java版本信息，可见java环境变量配置成功。

1.3下载安装hadoop

我下载的hadoop是来自清华的镜像：Index of /apache/hadoop/common (tsinghua.edu.cn) 版本是hadoop -3.2.3。

同样，在usr目录下创建hadoop文件夹，将下载好的hadoop压缩包移动至该文件中，进行解压

命令：sudo tar -zxvf hadoop-3.2.3.tar.gz

使用ls命令可查看解压后的文件，如下

1.4配置hadoop的环境变量

命令：sudo vim /etc/profile，添加如下环境变量

HADOOP_HOME=/usr/hadoop/hadoop-3.2.3

export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME

:wq，命令保存退出，输入hadoop version可见hadoop的环境变量配置成功！

1.5将hadoop和java绑定

编辑hadoop-env.sh配置文件，hadoop-env.sh 位于：/usr/hadoop/hadoop-3.2.3/etc/hadoop路径下，

修改：JAVA_HOME=/usr/java/jdk1.8.0_181。保存退出

1.6配置hosts列表和修改主机名。

hosts 列表作用是让集群中的每台服务器彼此都知道对方的主机名和 ip 地址。

可使用hostnamectl set-hostname master命令修改当前虚拟机的主机名。

root模式下 vim /etc/hosts ，添加主机名和ip。

其他的四台从属服务器，会在之后进行配置。

1.7关闭和禁用防火墙

关闭防火墙后我们可以在本机访问虚拟机的端口。

\6. 安装中遇到的问题及解决方案

6.1解压压缩包问题

解压压缩包要使用su命令进入root用户，或者使用sudo tar -zxvf + 压缩包名。在解压jdk和hadoop都遇到了此问题，原因是当前用户没有文件读写权限。

6.2编辑文件问题

在配置环境变量，修改etc/profile文件时，也要使用root用户或sudo vim + 文件名的命令，否则会无法保存，报错 E212: Can‘t open file for writing，当前用户没有编辑权限。

6.3初始化hadoop节点报错

原因是没有文件读写权限，使用命令：sudo chmod -R a+w /usr/hadoop/ 修改文件夹权限。
